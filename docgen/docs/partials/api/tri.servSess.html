servSess (service in module tri )

Description

Provides app session management based on the user login status. It is the single service that maintains overall application context.

Its functionality includes:

  • User inactivity timeout functions.
  • User authorization validation functions
  • Current Table context management
  • Menu function utilities
  • Miscellaneous utility functions

The Edit Controller and List Controller often reflect function calls in their scope to the matching calls in servSess.

  • useTable(scope, sTable, sel)

    This function is pivitol in setting the Triangular current Table context. It sets the internal oTAB variable.

    If sTable is null the scope parameter value is assigned to the existing oTAB object scope value.

    If sTable is not null an attempt to match sTable against the recTitle value in the array of registered tables (see registerTable) is made. if sucessful a new dynamic Table object is created from the static copy and assigned to oTAB. If a match is not made oTAB is set to null.

    If oTAB at this point is null a return of null is made.

    The scope.oSel value is updated with the existing oTAB.props.oSel value. A copy is preserved in the static copy so that settings and values are remembered across Table changes.

    The the oTAB.props.primeList function exists it is called.

    Parameters

    • scope – {Object} –

      current caller scope

    • sTable – {string} –

      table name (title) to find

    • sel – {Object} –

      previous/default selection (filter) context

    Returns

    {Object} –

    The current dynamic Table object.
