servTBL (service in module tri )

Description

Specific I/O services for TBL mode tables

Refer to Record Storage for a description of the record modes and to servCRUD for the API common functions.

Methods

  • insertRec(oTP, obj, oaa, bInter)

    Used internally to insert a record into the Data Object cache. If the oaa is null a new Data Object is inserted.

    When bInter is true the display updates are not invoked. Used to save partial updates to a record rather than lose the whole thing should the system crash.

    Parameters

    • oTP – {Object} –

      The Table.props cache where the object array is stored.

    • obj – {Object} –

      The Data Object to insert.

    • oaa – {Object} –

      The Object Access Address. A string or number determining the insert point. See Record Storage for a description of oaa. Note that the servTBL mode does not support chunked data.

    • bInter – {Boolean} –

      When true the update is intermediate with more to follow.

  • isCombined()

    Returns whether the service type combines the individual JSON objects into an array for storage and retrieval.

    Returns

    {Boolean} –

    Always returns false.

  • setObjs(oTP, objs)

    See servREC.setObjs since this function processes the call.

    Parameters

    • oTP – {Object} –

      The Table.props where the object array is stored.

    • objs – {Object[]} –

      The array of Data Objects to store.
