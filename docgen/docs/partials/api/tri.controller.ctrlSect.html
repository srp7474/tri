ctrlSect (controller in module tri )

Description

Provides a context managment scheme to know what context the infrastructure is in. For all cases the function is forwarded to the equivalent servSess function and where appropriate the return of that function is returned.

Usage

ctrlSect($scope, servSess, servGAE);

Parameters

  • $scope – {Object} –

    The scope of the controller

  • servSess – {Object} –

    The servSess service pointer

  • servGAE – {Object} –

    The servGAE service pointer

Methods

  • cancelNewPassword()

    Clears the sPasswordErr field

  • closeConfirmMsg()

    Clears any confirmation message.

    This call is forwarded to servSess.setConfirmMsg with a value of null.

    Returns

    {Object} –

    undefined

  • closeErrorMsg()

    Clears any error message.

    This call is forwarded to servSess.setErrorMsg with a value of null.

    Returns

    {Object} –

    undefined

  • getConfirmMsg()

    Returns any User contextual confirmation message.

    This call is forwarded to servSess.getConfirmMsg and returns the value obtained thereby.

    Returns

    {Boolean} –

    the value returned by servSess.getConfirmMsg

  • getErrorMsg()

    Returns any User contextual error message.

    This call is forwarded to servSess.getErrorMsg and returns the value obtained thereby.

    Returns

    {Boolean} –

    the value returned by servSess.getErrorMsg

  • hasConfirmMsg()

    Returns whether a User confirmation message exists.

    This call is forwarded to servSess.getConfirmMsg and returns whether the result is null (false) or not (true).

    Returns

    {Boolean} –

    the value returned by servSess.hasConfirmMsg != null

  • hasError()

    Returns whether sPasswordErr contains any data.

    Returns

    {Boolean} –

    whether sPasswordErr is null (false) or not null (true)

  • hasErrorMsg()

    Returns whether a User error message exists.

    This call is forwarded to servSess.getErrorMsg and returns whether the result is null (false) or not (true).

    Returns

    {Boolean} –

    the value returned by servSess.hasErrorMsg != null

  • hasSession()

    Returns whether a valid session exists. ie. The User is properly logged in and validated.

    This call is forwarded to servSess.hasSession and returns the value obtained thereby.

    Returns

    {Boolean} –

    the value returned by servSess.hasSession

  • isSect(sSect)

    Returns whether sect section matches the current section.

    This call is forwarded to servSess.isSect and returns the value obtained thereby.

    Parameters

    • sSect – {string} –

      value to match against

    Returns

    {Boolean} –

    the value returned by servSess.isSect

  • submitNewPassword()

    Used in concert with the HTML template 'apps-change-password.html' to submit a password change.

    If the password entered is valid (at least one alphabetic) and the confirm field matches the method servGAE.changePassword is called.

    If successful servSess.setConfirmMsg is called.

    If not successful servSess.Sess.#setErrorMsg is called.

Properties

  • sPasswordErr

    Initial password change validation error field.

    Set during a submitNewPassword method call.
