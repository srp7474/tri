Overview

Often in developing an application you need to add some functionality for which you suspect or know an example exists. This page is designed to be an aid to finding examples of working code in the Triangular code base.

Refer also to the Hints and Techniques page where other aids may be found, especially related to debugging techniques.

Example Code

The examples are broken into catagories to assist in finding them. Each points at the approximate location where it can be found. It may reference othe pieces of code elsewhere. If the explain link is present it points to some code that has supplementary documentation.

Filters

ConstructUsageReferenceExplained
Filter mask A filter mask is a simple filter that allows for a regular expression to match the contents of the subject field. To use the type:'mask' is used in the filters Table properties and the field property names the column to match against. Hello.js lines 80, 81 explained

The TableDonors.js filter is more realistic in that it uses the input data to determine what field to test. TablesDonors.js line 42 explained

AngularJS Subtleties

ConstructUsageReferenceExplained
Delayed $digest Sometimes an external event requires that you run a AngularJS $digest cycle when the event completes. The Triangular encounters this situation when a record read completes. The idiom
setTimeout(function()
    {$rootScope.$digest()},0);
causes the AngularJS $digest cycle to run immediately the current task completes. 		tri/Tri.js
Focus change Sometimes during a $digest cycle you wish to set the window focus to an input field. The plain setFocus on the input field will not do this because the interface is too unstable running the $digest cycle. A variation on the previous idiom of
setTimeout(function()
    {setCustFocus(...)},0);
where setCustFocus is the custom routine to call will set the focus when the interface is stable. 		Chqs.js line 272 and others explained
Use of $eval Sometimes it is necessary to postpone evaluations until late in the game. This is particuarly true with code in the HTML pages that is handled by directives. The AngularJS $eval method was found to achieve this postponement. tri/AppsBase.htm apps-lister.html template
Hidden input field Sometimes an event needs to trigger a UI update in a complex page. A <input type=hidden ng-model=xxx can be used. The xxx model is updated with a unique value such as one that depends on the timer when the UI page needs to be refreshed. Be careful not to update the model too many times.

Triangular uses this to update the Lister after a record is dropped. Refer to dropTableRecord in "tri/TRI.js" and apps-lister.html in "tri/AppsBase.htm". 		tri/TRI.js

Datastore Topics

ConstructUsageReferenceExplained
REC Data The REC Mode stores individual records are grouped together under a single key. It is implemented by specifying servRec: servREC in the Table Definition. Hello.js 38 explained
TBL Data The TBL Mode stores individual records each with their own key. It is implemented by specifying servRec: servTBL in the Table Definition. Deps.js 98 explained
Chunked Data The REC Mode can be stored as chunks when there are a large number or records It is implemented by adding a % at the end of the recName property while still specifying servRec: servREC in the Table Definition. TableDonors.js line 22 explained
Unique Keys By default a fields value does not have to be unique. Adding property setting many:false to the Col:many causes Triangular validation routines to insist that a column's field value be unique. Deps.js line 106 explained
Intermediate Saves During a long Edit operation, especially if there are intermediate lines being added, it is desirable that the results be stored so that in the event of a crash the usre's work is not lost. The TBL Mode insertRec supports this. See servTBL.insertRec. Note that means of a rollback if necessary will have to be programmed into the application. Deps.js 397 explained
Variable Keys The key of the Table to use sometimes needs to be arrived at progromatically. The "Chqs" app has such a need. The fileKey and fileKeyRef properties in the Table Definition are used. Chqs.js line 101 explained
Dynamic Key Names The key under which a TBL Mode record is stored sometimes needs to be arrived at progromatically. The "Deps" app has such a need. The createExit Table Definition is used. In that exit the value is computed and the oObj.KeyName special field name set with the computed value. This will be used by Triangular to store the values. Deps.js line 614 explained
