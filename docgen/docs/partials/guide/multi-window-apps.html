Overview

AngularJS, in their own documentation AngularJS introduction, is purportedly best at single-page apps. While this is true it is easy to extend AngularJS to support multiple pages using a couple of patterns.

The Display/Form Pattern

The Display Page Design and Form Page Design describe how the Form Page Design can be used as secondary pages in an appliocation the uses Display Page Design as the primary page.

The "Deps.htm" and "Chqs.htm" are samples of this pattern.

The Admin Page Pattern

With this pattern the logged in User token is shared among several application pages so that validateToken of secondary pages return a 'logged in' status. This requires that the setCust parameter is the same for all pages.

In this pattern either the URL or the localStorage is used to maintain the 'breadcrumbs' so that the secondary page knows how to return to the primary page.

The "Admin.htm" page is an example of a page that acts as the secondary page. In its case the app name is passed to as a URL parameter so that it can be used as the secondary page for any application.
