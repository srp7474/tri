Steps 04 Prepare MyApp Data and 05 Code MyApp go hand in hand.

If your app will generate the data through the WebPage then you can proceed to 05 Code MyApp.

If your app requires external data to be added then the steps on this page are required.

Note The WAKS Utility will have to be adjusted in a non-windows platform.

Uploading data to the App

  1. The WAKS Utility is used to move batches of data to and from your app.

  2. Before proceeding let us experiment with uploading data to the present MyApp application. We know its roots are the Hello App so we will use the data conveniently stored with the Development Package Demonstration Data

  3. Start a command line window and make the CWD x:\tri-devp\bats as defined in Development Package Preparation

  4. Issue command waks set myapp super super and you will see

    configured WAKS: -cust myapp -user super -pw super

    This sets the app id and the SuperUser credentials.

  5. Issue command waks dir ..\data and you will see

    configured WAKSDIR: ..\data

    This sets the working directory.

  6. In folder x:\tri-devp\data copy and rename msgs-msg-messages.json.txt to myapp-msg-messages.json.txt

  7. Issue command waks ship % % and you will see it refuses to upload the data.

  8. Issue command waks ship % % -r and you will see it uploads and replaces the data.

  9. Verify this by refreshing the MyApp app in the browser and pressing List Messages Records. You will see your previous data deleted and the replacement messages.
