MyApp is now ready to upload to the GAE Site. These are the steps. Modify for a non-windows platform.

GAE Site Upload

  1. Start a command line window and make the CWD x:\tri-devp\bats as defined in Development Package Preparation

  2. Type appcfg to initiate shipping the application to the GAE Site.

  3. You will be prompted for Email:. Enter Google account (your email without @gmail.com suffix)

  4. You will be prompted for Password:. Enter the password associated with this gmail account.

  5. Note: On subsequent uploads you may not be prompted for this information as for a period of time it remembers.

  6. It will crunch for a few seconds until it announces

    Update for server default completed successfully

GAE Site Datastore Priming

Note: This will only be necessary the first time you upload MyApp to the GAE Site.

  1. As before the Datastore is unprimed.

  2. Point your browser at URL tri-test.appspot.com (where tri-test is your GAE Site ID) and you should see the familiar screen.

    Note: An intermediate Prime.htm screen may show for a few seconds while the site warms up. Be patient.

  3. Repeat the procedure starting at point 2 in Configuring the Datastore remembering to use URL tri-test.appspot.com rather than localhost:8882

Upload App data to GAE Site

  1. If there is data to upload proceed as described in Uploading data to the App. But first read on.

  2. Before using the upload facilities you have to tell the WAKS Utility to use the production site (GAE Site).

  3. First issue the command waks site tri-test where tri-test is the MyApp site id.

  4. Then prefix prod to each WAKS Command. Thus the sequence

    waks set myapp super-userid super-password
    waks dir ..\data
    waks prod ship % %

    would ship the myapp data stored in the ..\data folder
