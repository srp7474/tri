Let us assume at this point that MyApp is complete and working.

Theoretically, it could be hosted via an intranet or even globally by making your internal server running this code accessible to the internet. This may not be ideal for the following reasons:

  • The industrial strength of the development server provided by Google is questionable. It is provided as a development tool.

  • Limited to a single machine serving

  • There is no built in data redundancy like that provided with the Google App Engine Site.

To move MyApp to the Google App Engine Site. the following steps are required

Get GAE Site ID

  1. If you do not yet have one get a Google Account from Google Account Login

  2. Logon to your Google Account using Google Account Login

  3. Logon to AppEngine from this window. Use the link https://appengine.google.com/

  4. If you already have applications they will listed on the screen.

  5. Press the Create Application button.

  6. Create an Application Identifier: that is acceptable. This tutorial uses tri-test. Change as appropriate. Press the Create Application button. You will the see Application Registered Successfully meaning the name chosen has been registered.

Insert GAE Site ID into your local setup

CAUTION: The following steps will cause the local server to trash your local_db.bin mentioned in Datastore. If you have data you care about use the WAKS Utility to preserve it now. You can also rename local_db.bin and datastore-indexes-auto.xml with the intent of retrieving the data later by switching back to <application>triapps</application> and using WAKS Utility at that time.

  1. Use <Ctrl-C> to stop the webserver started in Starting the Local WebServer

  2. Edit appengine-web.xml in x:\tri-devp\war\WEB-INF and change triapps in line <application>triapps</application> to you new GAE Site ID. In our example this would be tri-test.

  3. As additional security, you can modify the file x:\tri-devp\war\WEB-INF\app-cita\cita.ace

    In about line 16 of the cita.ace file change the string value of PasswordObfuscator= to a different value. This string is used in converting the password into a numeric token. A value known only to you makes the cracking of the computed token value much more difficult.

  4. Restart the webserver per Starting the Local WebServer

Configuring the Datastore

At this point the Datastore will be unprimed and will need to be primed as detailed in Datastore Prime.

  1. In the Browser goto URL localhost:8882

  2. You should see The Triangular Datastore requires priming screen

  3. Create the SuperUser as instructed. It is suggested you use a value different from the demo value of super.

  4. Login as the SuperUser using the credentials from point 3.

  5. You will be requested to add Add Application AdminUser. Do this using the procedure already used in Configure the Datastore starting from point 2. There will be no other apps present.

  6. Add other User records as required by MyApp using the Configure the Datastore point 7.

  7. If you have data to add to MyApp do so now using the WAKS Utility and as previously covered in Prepare MyApp Data. Remember to use the SuperUser credentials from point 3.
